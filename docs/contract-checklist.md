# dllm 訂閱合約 — AI 撰寫合約注意事項

> 此文件為提供 AI 撰寫合約時的指示清單，確保合約涵蓋 dllm 商業模式中的特殊條款。

## 一、商業模式摘要（提供給 AI 的背景）

```
產品：dllm（LLM 執行環境，取代 Ollama）
月費：NTD 10,000（含軟體授權 + 硬體借用）
硬體選項：
  - 標準方案：Mac Mini M4 Pro 64GB（免費借用）
  - 升級方案：DGX Spark 128GB / ASUS GB-10 / 銘凡 GB-10（客戶付硬體差價）
硬體所有權：歸我方所有，非賣斷
授權方式：月租訂閱制，非永久授權
底層引擎：vLLM（開源，Apache 2.0）
核心程式：Rust 開發，我方擁有完整著作權
```

## 三、常見陷阱與注意事項

### 3.1 不要用標準 SaaS 合約模板

一般 SaaS 合約預設：
- 資料存在雲端 → 你是本地部署，資料在客戶設備
- 服務中斷責任 → 你的服務是本地跑，不依賴你的伺服器
- 軟體授權永久或年約 → 你是月租，且含硬體

**請 AI 務必從頭起草，不要套 SaaS 模板。**

### 3.2 硬體折舊與會計處理

這不是法律條款，但客戶的財務部門可能會問：

| 項目 | 我方立場 | 說明 |
|------|---------|------|
| 硬體折舊 | 我方認列 | 設備所有權在我方，折舊由我方攤提 |
| 月費認列 | 客戶列為費用 | 客戶支付月費列為營業費用 |
| 硬體賠償 | 客戶列為損失 | 若客戶損壞設備，賠償金列為客戶損失 |

### 3.3 降級模式的說明要寫清楚

一般客戶不會理解「降級模式」是什麼意思。合約中應明確說明：

> 「降級模式：API 端點（/health、/v1/models）仍可正常回應，但模型推理（/v1/chat/completions）將回傳錯誤。客戶的整合開發環境不會完全斷線，但無法產生新的 LLM 回覆。」

### 3.4 合約期間與自動續約

建議：
- **首次合約**：一年期（讓客戶有承諾感）
- **續約**：自動按月續約（降低摩擦）
- **終止**：任何一方可於 30 日前書面通知終止

### 3.5 適用法律與管轄

- 建議約定：**中華民國法律**為準據法
- 管轄法院：客戶所在地或雙方合意之法院
- 若客戶為外商在台分公司，仍建議用中華民國法律
